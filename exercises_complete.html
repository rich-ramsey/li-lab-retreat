Key tidyverse verbs

These are the main functions you will need today. If you get stuck, the tidyverse cheat sheets are a good first stop.

Function Package What it does
pivot_longer() tidyr Reshapes data from wide to long format
mutate() dplyr Creates or modifies columns
filter() dplyr Subsets rows by a condition
select() dplyr Subsets columns
summarise() dplyr Aggregates data, e.g. computing means

Note: you may see group_by() paired with summarise() in older code and cheat sheets. The modern approach uses the .by argument inside summarise() directly, which is what we use here.

Pipes and layers: the same idea twice

Two bits of syntax will do a lot of work today.

The pipe |> chains data through a sequence of steps. Each step takes the output of the previous one and passes it to the next function. Think of it as “and then”:

data |>
  filter(group == "embodied") |>
  summarise(mean_correct = mean(correct), .by = pid)

The plus sign + in ggplot2 works the same way, but for building plots. Each + adds a new layer on top of the previous one:

ggplot(data, aes(x = group, y = correct)) +
  geom_violin() +
  geom_jitter() +
  labs(x = "Group", y = "Proportion correct")

Same mental model, different context: one builds a data pipeline, one builds a plot.

Exercise 1: Wide to long

The item columns (item_1, item_17, etc.) each represent one observation of the same variable: whether a participant got an item correct or not. In tidy data, each variable has its own column. So these 8 columns should become two: one for item identity, one for the response.

Your job is to reshape the data from wide to long format using pivot_longer(). The result should have one row per participant per item, with columns pid, group, pre_score_z, item, and correct.

dat_simple_long <- dat_simple_wide |>
  pivot_longer(
    cols = starts_with("item_"),
    names_to = "item",
    values_to = "correct"
  )

Exercise 3: Single-level model

Fit a single-level logistic regression predicting correct from group and pre_score_z. This model ignores the fact that observations are nested within participants.

Use glm() with family = binomial. Use tidy() from broom.mixed to view the output, with conf.int = TRUE to get confidence intervals.

m_single <- glm(
  correct ~ group + pre_score_z,
  data = dat_simple_long,
  family = binomial
)

Exercise 4: Multilevel model

Now fit a multilevel logistic regression using glmer() from the lme4 package. Add a varying intercept for each participant. This allows each person to have their own baseline response probability, over and above the group average.

A note on convergence: lme4 fits models using maximum likelihood estimation, and multilevel models — especially with binary outcomes — frequently fail to converge. This is not a bug; it reflects a genuine limitation of the algorithm. Bayesian software such as brms fits the same models via a different algorithm that is more robust and does not have this problem, which is why many researchers prefer it for multilevel modelling. For the models we are fitting today, lme4 should be fine.

m_multi <- glmer(
  correct ~ group + pre_score_z + (1 | pid),
  data = dat_simple_long,
  family = binomial
)
